Flutter页面路由与页面曝光技术实现

android flutter 埋点 路由
Flutter
发布日期:   2023-01-17

前言

在移动端开发中，数据埋点是一种常用的数据采集方法，方便产品/运营系统性的统计分析复杂的用户数据。我们在App端所设置的自定义事件：点击，访问页面，通过数据埋点的方式进行记录。目前市面上原生app埋点统计SDK已经很成熟，如百度统计SDK,神策埋点SDK,友盟统计等。今天我们旨在分析原生与flutter混合体系下flutter路由与页面曝光的实现方案。

一 Flutter页面路由简介

1.1 Flutter路由简介

在Flutter中一个页面对应一个Route(路由)，不同的页面之间进行跳转和回退均涉及到路由的切换，简单来说Flutter中的一个页面（路由）类似Android中的一个Activity，iOS中的一个ViewController。Flutter中页面路由的管理和切换交由Navigator类（所有路由的根）来管理。

  1. 打卡新的页面：Navigator.of(context).push(route)

  2. 关闭当前页面: Navigator.of(context).pop()

1.2 Flutter路由监听

上文提到Flutter中通过Navigator来管理Route路由,同时也提供了RouteObserver类和注册的方法，用于监听Flutter侧页面的变化，如MaterialApp提供了navigatorObservers属性， 用于注册一系列路由监听器，在NavigatorObserver内部维护了一个List<_RouteEntry> _history = <_RouteEntry>[] 栈，来记录flutter所有已打开的路由

class MyApp extends StatelessWidget {
   @override
   Widget build(BuildContext context) {
     return MaterialApp(
       initialRoute: _chooseInitialRouteName(),
       //routes: <String,WidgetBuilder>{
       //"/Home":(context) => const HomeRoute(),
       //"/Login":(context) => const LoginRoute(),
       //"/Themes":(context) => const ThemeChangeRoute(),
       //"/Language":(context) => const LanguageRoute(),
       //"/Webview":(context) => WebViewRoute(url: ModalRoute.of(context)!.settings.arguments as String,),
      // "/Test":(context) =>  TestRoute(),
     //},
       onGenerateRoute: _generateRoute,
       navigatorObservers: [ CustomRouteObserver(),],
     );
   }

   String _chooseInitialRouteName() {
     // 获取原生app传递的初始化路由
     var platformInitRoute = PlatformDispatcher.instance.defaultRouteName;
     var initRoute = platformInitRoute;
     if (platformInitRoute.isEmpty || platformInitRoute == "/") {
     initRoute =  "/Home";
     }
     debugPrint("+++++++++++++++++RouteTag _chooseInitialRoute() initRoute=$initRoute,platformInitRoute=$platformInitRoute+++++++++++++++++");
     return initRoute;
   }

   Map<String,WidgetBuilder> initPageRoute() {
     return {
       "/Address": (context) => const AddressRoute(),
       "/Home": (context) => const MyHomePage(title: "this is home"),
       "/Test": (context) => const TestPageRoute(),
     };
   }
   /// 路由拦截，修改路由参数
   Route<dynamic>? _genPageRoute(RouteSettings settings){
     String routeName = settings.name ?? "";
     Map<String,WidgetBuilder> pageRoutes = initPageRoute();
     var builder = pageRoutes[routeName];
     if (routeName == "/") {
       debugPrint("+++++++++++++++++RouteTag _genPageRoute() start '/'+++++++++++++++++");
       return null;
     }
     if (builder == null) {
       builder = (context) => UnKnownPageRoute(title: settings.name ?? "unknown route",);
       debugPrint("+++++++++++++++++RouteTag _genPageRoute() 未注册${settings.name}+++++++++++++++++");
     }else {
       debugPrint("+++++++++++++++++RouteTag _genPageRoute() take${settings.name}+++++++++++++++++");
     }
     var copySettings = settings;
     if (routeName == "/Test") {
       var args = (settings.arguments ??  {}) as Map;
       args["arg1"] = "动态添加参数1";
       args["arg2"] = "动态添加参数2";
       copySettings = settings.copyWith(name: settings.name,arguments: args);
     }else if (routeName == "/Home") {
       var args = (settings.arguments ??  {}) as Map;
       args["path"] = "home";
       args["arg2"] = "动态添加参数2";
       copySettings = settings.copyWith(name: settings.name,arguments: args);
     }
     return MaterialPageRoute(builder: builder,settings: copySettings);
   }

   /// 未知路由兜底处理
   Route<dynamic> _genUnKnownPageRoute(RouteSettings settings){
     debugPrint("+++++++++++++++++RouteTag _genUnKnownPageRoute() 未注册${settings.name}+++++++++++++++++");
     return MaterialPageRoute(builder: (BuildContext context){
       return UnKnownPageRoute(title: settings.name ?? "unknown route",);
     });
   }
}

自定义路由监听器CustomRouteObserver继承自RouteObserver，RouteObserver继承自NavigatorObserver，NavigatorObserver提供了几个方法来监听页面路由的变化

  • void didPush(Route route, Route? previousRoute) : 该方法表示当有新页面打开时，新的route，以及旧的previousRoute

  • void didPop(Route route, Route? previousRoute)：该方法表示当有页面退出栈顶

  • void didReplace({Route? newRoute, Route? oldRoute})：该方法表示当旧的oldRoute被newRoute替换

  • void didRemove(Route route, Route? previousRoute)：该方法表示route被移除

  • void didStartUserGesture(Route route, Route? previousRoute)：该方法表示用户开始使用手势返回

  • void didStopUserGesture()：该方法表示用户结束手势返回

因此，这里我们重点关注didPush和didPop两个方法即可监听到所有的flutter路由的打开和退出

需要注意的是，在flutter中路由分为页面路由（PageRoute，如：MaterialPageRoute，CupertinoPageRoute）

和模态弹窗路由（PopupRoute，如： Dialog, bottomSheet,ContextMenu,DropDownRoute,PopupMenu），二者都继承自ModalRoute，在做页面曝光的时候，我们更关心页面路由，而非模态弹窗路由，因此需要区分路由的类型进行判断和筛选

关于MaterialApp中路由属性使用优先级的说明：

initialRoute，

home，

routes，

onGenerateRoute，

onUnknownRoute

  • 如果既没有设置home，又没有设置initialRoute，则初始路由为”/“

  • 如果既设置了home,又设置了initialRoute，以home为主

  • 如果设置了home,则不能再注册 “/“ 对应的路由，会冲突，运行时崩溃，home 等价于”/“代表的路由

  • 如果既设置了routes，又设置了onGenerateRoute，则优先使用routes,如果在routes找不到，才会去onGenerateRoute中查找

  • 如果既没有设置home,又没有设置initialRoute，则初始路由为 “/“，会依次从routes，onGenerateRoute中查找”/“对应的路由，onGenerateRoute可以对”/“返回null，则不压入栈，返回非null，则压栈

  • 如果routes，onGenerateRoute中都找不到对应的路由，则会走onUnknownRoute进行处理，如果未设置onUnknownRoute，则运行时崩溃

  • 如果相对路由拦截，及修改路由参数RouteSettings.arguments，则只能在onGenerateRoute中处理

优先级：

home > initialRoute

routes > onGenerateRoute > onUnknownRoute

因此，如果有拦截路由修改参数的需求，建议使用initialRoute+onGenerateRoute+onUnknownRoute，并且在onGenerateRoute处理”/“返回null

二 Flutter页面曝光技术实现

基于第一节中关于Flutter页面路由的知识和原理，我们可以通过自定义路由监听器CustomRouteObserver实现对页面打开和关闭的监听，从而间接实现和原生app类似的页面曝光。

注意：在Flutter中打开新路由页面，建议使用命名路由的方式，以来方便统一在MaterialApp中注册和管理路由，二来方便实现页面路由和页面埋点之间的映射关系

2.1 页面曝光的实现

  • 当void didPush(Route route, Route? previousRoute)回调时，我们获取到的route即为新打开的页面，触发新页面的前进可见曝光

  • 当void didPop(Route route, Route? previousRoute) 回调时，我们获取到的previousRoute即为旧页面的回退可见曝光

  • 在获取的route中我们筛选出页面类型的PageRoute，记为有效的页面曝光，过滤模态弹窗路由

  • 通过获取Route的name（route.settings.name，”/“表示flutter的初始路由页面）和页面曝光埋点进行映射，即可生成当前路由页面的曝光事件

整体实现如下：

import 'package:flutter/cupertino.dart';
import 'package:flutter/material.dart';
import 'package:flutter/services.dart';

/// 1.自定义MaterApp路由监听器，通过监听flutter路由切换，来实现flutter app 页面曝光
/// 2. 路由包含了PopupRoute(模态弹框路由：如 dialog，BottomSheet)，PageRoute（页面路由，我们真正需要关心的部分）
/// 3.通过路由的名称和页面埋点进行映射，即可实现页面曝光埋点，Map<Sting,String> = {route.settings.name : 页面埋点}
/// 4.如果页面曝光需要扩展字段，则通过路由参数传递，route.settings.arguments
class CustomRouteObserver<R extends Route<dynamic>> extends RouteObserver<R>{

  final List<Route>_routeHistory = [];
  
  @override
  void didPush(Route route, Route? previousRoute) {
    super.didPush(route, previousRoute);
    _routeHistory.add(route);
    debugPrint("--------------------------start----------------------------------");
    debugPrint('didPush route: ${route.settings.name} +++ route.hashCode=${route.hashCode} +++ route.isActive=${route.isActive} +++ route.runTimeType=${route.runtimeType}； '
    ' previousRoute:${previousRoute?.settings.name} +++ previousRoute.hashCode=${previousRoute.hashCode} +++ route.isActive=${previousRoute?.isActive}');
    var routeType = route.toString();
    var routeRuntimeType = route.runtimeType;
    debugPrint("routeType:$routeType,------- routeRuntimeType:$routeRuntimeType");
    
    // 如果当前push的路由是页面类型，则生成当前被push页面的didAppear事件
    var isCurPageRoute = _isPageRoute(route);
    if (isCurPageRoute) {
      _pageDidAppear(route);
    }
    // 如果当前被push的路由是类型类型，前一个路由也是页面类型，则生成前一个页面的disappear事件，排除模态弹框的影响
    if (isCurPageRoute && _isPageRoute(previousRoute)) {
      _pageDisappear(previousRoute!);
    }
    debugPrint("---------------------------end---------------------------------");
  }

  @override
  void didPop(Route route, Route? previousRoute) {
    super.didPop(route, previousRoute);
    _routeHistory.remove(route);
    debugPrint('didPop route: ${route.settings.name} +++ route.hashCode=${route.hashCode}；  previousRoute:${previousRoute?.settings.name} +++ previousRoute.hashCode=${previousRoute.hashCode}');
    // 当前pop的路由是页面类型，则生成当前被pop页面的disappear事件
    var isCurPageRoute = _isPageRoute(route);
    if (isCurPageRoute) {
      _pageDisappear(route);
    }
    // 如果当前被pop的路由是页面类型，前一个路由是页面类型，则生成前一个页面的didAppear事件，排除模态弹框的影响
    if (isCurPageRoute && _isPageRoute(previousRoute)) {
      _pageDidAppear(previousRoute!,pageBackward : true);
    }
  }
  
  @override
  void didReplace({Route? newRoute, Route? oldRoute}) {
    super.didReplace(newRoute:newRoute, oldRoute: oldRoute);
    debugPrint('didReplace newRoute: ${newRoute?.settings.name} +++ newRoute.hashCode=${newRoute.hashCode}；oldRoute:${oldRoute?.settings.name} +++ oldRoute.hashCode=${oldRoute.hashCode}');
  }
  
  @override
  void didRemove(Route route, Route? previousRoute) {
    super.didRemove(route, previousRoute);
    debugPrint('didRemove route: ${route.settings.name} +++ route.hashCode=${route.hashCode}；previousRoute:$previousRoute +++ previousRoute.hashCode=${previousRoute.hashCode}');
  }
  
  @override
  void didStartUserGesture(Route route, Route? previousRoute) {
    super.didStartUserGesture(route, previousRoute);
    debugPrint('didStartUserGesture route:${route.settings.name} +++ route.hashCode=${route.hashCode}；previousRoute:$previousRoute +++ previousRoute.hashCode=${previousRoute.hashCode}');
  }
  
  @override
  void didStopUserGesture() {
    super.didStopUserGesture();
    debugPrint('didStopUserGesture');
  }
  

  /// 判断路由类型是否为PageRoute
  bool _isPageRoute(Route? route) {
    var isPageRoute = true;
    if (route == null) {
      isPageRoute = false;
    }
    // ModalRoute：like Dialog, bottomSheet,ContextMenu,DropDownRoute,PopupMenu
    if (route is PopupRoute) {
      isPageRoute = false;
      debugPrint("route is PopupRoute");
    }
    if (route is MaterialPageRoute) {
      debugPrint("route is MaterialPageRoute");
    }
    if (route is CupertinoPageRoute) {
      debugPrint("route is CupertinoPageRoute");
    }
    if (route is PageRouteBuilder) {
      debugPrint("route is PageRouteBuilder");
    }
    return isPageRoute;
  }
  
  
  /// 页面曝光埋点
  ///
  void _pageDisappear(Route route) {
    var pageName = route.settings.name;
    if ("/" == pageName) {
      pageName = "首页";
    }
    debugPrint('-------------------------- $pageName disappear() --------------------------');
  }

  /// 页面pause埋点
  ///
  void _pageDidAppear(Route route,{bool pageBackward = false}) {
    var pageAppearType = pageBackward ? "页面回退" : "页面前进" ;
    var pageName = route.settings.name;
    if ("/" == pageName) {
      pageName = "首页";
    }
    debugPrint('-------------------------- $pageName didAppear() 曝光类型：$pageAppearType--------------------------');
  }
  
  /// 原生容器退出时，最后一个路由不会触发pop操作，需要手动记录一次页面pop
  void _callViewControllerWillDestroy() {
    if (_routeHistory.isEmpty) {
      return;
    }
    Route route = _routeHistory.removeLast();
    if (_isPageRoute(route)) {
      _pageDisappear(route);
    }
  }
    
  /// 原生app生命周期监听
  Future<dynamic> nativeAppLifecycleRegistry(MethodCall call) async{
    if (call.method == "callViewControllerWillDestroy") {
      _callViewControllerWillDestroy();
    }
  }

}

三 原生app与Flutter混合体系下页面曝光技术实现

3.1 原生app与Flutter混合体系的区别

  • 原生app与Flutter混合体系下，以原生app为主，flutter作为内容依附在原生容器中，在Android中对应的原生容器为FlutterActivity,在iOS中对应的原生容器为FlutterViewController。

  • 混合体系下，每次打开新的FlutterActivity，都会新开FlutterEgine,会拖慢flutter的启动，因此可能会使用到预热和缓存的FlutterEngine，但是使用缓存的FlutterEngine一但已经初始化并制定了初始化路由，
    后续再打开flutter页面时无法再指定特定路由跳转（因此必然先达到初始路由页面，产生无效的页面曝光），因此为了解决这个问题，我们将会使用到FlutterEngineGroup

  • FlutterEngineGroup 用于管理多个FlutterEngine实例，并且可以定制化Flutter初始化入口，从而可以实现跳转flutter指定路由

  • 关于flutter多实例预热与缓存使用：https://flutter.cn/docs/development/add-to-app/android/add-flutter-screen?tab=prewarm-engine-kotlin-tab

  • Native-Flutter混合开发单引擎-多引擎模式区别

class BaseFlutterActivity : FlutterActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        // 绑定FlutterEngine 和methodChannle
        getFlutterEngineBindings().attach()
    }


    override fun onDestroy() {
        super.onDestroy()
        // 通知flutter端，原生容器即将销毁
        getFlutterEngineBindings().methodChannel.invokeMethod("callViewControllerWillDestroy",null)
        getFlutterEngineBindings().detach()
    }

    override fun provideFlutterEngine(context: Context): FlutterEngine? {
        // 使用定制化的FlutterEngine
        return getFlutterEngineBindings().engine
    }

    fun getFlutterEngineBindings(): EngineBindings
}

EngineBindings 类，主要处理dart函数入口的定制，和MethodChannel的实现

package cn.hipac.hello.compose.ui

import android.app.Activity
import android.content.Intent
import android.view.View
import android.widget.Toast
import io.flutter.FlutterInjector
import io.flutter.embedding.engine.FlutterEngine
import io.flutter.embedding.engine.dart.DartExecutor
import io.flutter.plugin.common.MethodChannel

class EngineBindings(activity: Activity,entryPoint: String,initialRoute: String? = null, vmArgs: List<String>? = null) {
  val methodChannel: MethodChannel
  val engine: FlutterEngine
  val app: App

    init {
        app = activity.applicationContext as App
        val dartEntrypoint = DartExecutor.DartEntrypoint(FlutterInjector.instance().flutterLoader().findAppBundlePath(),entryPoint)
        val options = FlutterEngineGroup.Options(activity).apply {
            // 指定dart 入口函数
            this.dartEntrypoint = dartEntrypoint
            // 指定flutter初始路由
            this.initialRoute = initialRoute
            // 指定dart入口函数参数
            this.dartEntrypointArgs = vmArgs
        }
        //engine = app.flutterEngineGroup.createAndRunEngine(activity,dartEntrypoint)
        engine = app.flutterEngineGroup.createAndRunEngine(options)
        // 注册 native-flutter 通信通道 
        methodChannel = MethodChannel(engine.dartExecutor.binaryMessenger, channelName)
    }

    fun attach(){
        methodChannel.setMethodCallHandler { call, result ->
            when (call.method) {
                "navigator" -> {
                    val res = navigator(TestActivity::class.java)
                    result.success(res)
                }
                "showToast" -> {
                    showToast("flutter call native toast method")
                    result.success(true)
                    app.getTopActivity()?.findViewById<View>(android.R.id.content)?.postDelayed({
                        methodChannel.invokeMethod("showFlutterToast","native call flutter toast method")
                    },2000L)
                }
                "addNewAddress" -> {
                   val res = call.arguments as Map<String,String>
                   showToast("新增地址：${res.toString()}")
                   result.success(true)
                }
                else -> {
                    result.notImplemented()
                }
            }
        }
    }

    fun detach() {
        engine.destroy()
        methodChannel.setMethodCallHandler(null)
    }

    private fun navigator(target: Class<out Activity>) : Boolean {
        val topActivity = app.getTopActivity()
        topActivity?.startActivity(Intent(topActivity,target))
        return true
    }

    private fun showToast(msg: String){
        Toast.makeText(app,msg, Toast.LENGTH_SHORT).show()
    }


}

3.2 原生app与Flutter混合体系下Flutter页面曝光的异常处理

  • 由于FlutterActivity作为原生容器承载一个或者多个flutter页面，当有多个flutter页面时，进行物理返回，会依次触发flutter路由的pop事件，直到最后一个flutter路由，此时back会直接先触发原生容器的onDestroy,
    而不会触发flutter的pop事件(但是会触发Flutter AppLifecycleState.detached)，因此会丢失最后一个页面的离开事件，解决方式是在原生容器销毁前通知flutter侧(或者在监听到AppLifecycleState.detached时进行记录)进行捕获最后一次pop事件
 
class BaseFlutterActivity : FlutterActivity() {
  override fun onDestroy() {
    super.onDestroy()
    // 通知flutter端，原生容器即将销毁
    getFlutterEngineBindings().methodChannel.invokeMethod("callViewControllerWillDestroy",null)
    getFlutterEngineBindings().detach()
  }
}


class CustomRouteObserver<R extends Route<dynamic>> extends RouteObserver<R>{
  /// 原生容器退出时，最后一个路由不会触发pop操作，需要手动记录一次页面pop
  void _callViewControllerWillDestroy() {
    if (_routeHistory.isEmpty) {
    return;
  }
  Route route = _routeHistory.removeLast();
  if (_isPageRoute(route)) {
    _pageDisappear(route);
  }
  }
  
  /// 原生app生命周期监听
  Future<dynamic> nativeAppLifecycleRegistry(MethodCall call) async{
    if (call.method == "callViewControllerWillDestroy") {
      _callViewControllerWillDestroy();
    }
  }
}

3.3.1 关于自定义Flutter dart入口函数的说明：

native侧，需要在FlutterEngine初始化时传入:

  • dartEntryPoint (方法名)
  • initialRoute(初始路由)
  • dartEntrypointArgs(入口函数参数，List)

flutter侧：
使用@pragma(‘vm:entr y-point’)标记入口函数，参数为List类型，可空

并且调用runApp()方法

3.3.2 关于自定义Flutter页面路由的定义：
  • 建议flutter侧MaterialApp中使用initialRoute属性初始化路由，而不用home属性,建议使用非”/“的有意义的路由名作为初始化路由（”/“对页面曝光监听路由有影响,无法识别具体的路由页面）
  • native侧需要与flutter侧注册的route 对应起来，如：native侧初始化路由为”/Address”, flutter侧也需要注册为”/Address”不能是”Address”
  • 如果flutter侧为提供onUnknownRoute实现，运行时会崩溃
3.3.3 关于自定义Flutter页面曝光埋点的获取方式：

  • 通过埋点配置文件（json文件）进行路由名称-页面曝光埋点进行映射，打包到app，由原生app提供注入，可能需要提前初始化（保证在第一个flutter页面被push之前）， 否则错过了曝光时机，获取不到数据，另外页面曝光的扩展字段可能需要额外方法实现注入

  • 在onGenerateRoute中拦截路由，将页面埋点数据注入到RouteSetting.arguments, 可以支持页面埋点动态添加业务数据 。路由参数RouteSettings.arguments格式固定为Map<String,dynamic>
