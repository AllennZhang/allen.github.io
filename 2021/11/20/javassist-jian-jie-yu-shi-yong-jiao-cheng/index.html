Javassist简介与使用教程

AOP 字节码 Java
Java
发布日期:   2021-11-20

Java 字节码以二进制的形式存储在 .class 文件中，每一个 .class 文件包含一个 Java 类或接口。Javaassist 就是一个用来 处理 Java 字节码的类库。它可以在一个已经编译好的类中添加新的方法，或者是修改已有的方法，并且不需要对字节码方面有深入的了解。同时也可以去生成一个新的类对象，通过完全手动的方式。

1 引用Javassist类库

  • Maven 方式

    <dependency>
  <groupId>org.javassist</groupId>
  <artifactId>javassist</artifactId>
  <version>3.27.0-GA</version>
</dependency>

  • Gradle 方式

repositories {
    mavenCentral()
}

dependencies {
    implementation 'org.javassist:javassist:3.27.0-GA'
}

2 Javassist主要API

  • 在 Javassist 中，类 Javaassit.CtClass 表示 class 文件。一个 CtClass (编译时类）对象可以处理一个 class 文件，ClassPool是 CtClass 对象的容器。它按需读取类文件来构造 CtClass 对象，并且保存 CtClass 对象以便以后使用

  • ClassPool

    • ClassPool 会在内存中维护所有被它创建过的 CtClass
    • getDefault : 返回默认的ClassPool 是单例模式的，一般通过该方法创建我们的ClassPool
    • appendClassPath, insertClassPath : 将一个ClassPath加到类搜索路径的末尾位置 或 插入到起始位置。通常通过该方法写入额外的类搜索路径，以解决多个类加载器环境中找不到类的尴尬
    • get , getCtClass : 根据类路径名获取该类的CtClass对象，用于后续的编辑

  • CtClass

    • removeMethod： 移除方法

    • addMethod： 添加方法

    • removeField：移除成员变量

    • addField： 添加成员变量

    • freeze : 冻结一个类，使其不可修改

    • isFrozen : 判断一个类是否已被冻结

    • prune : 删除类不必要的属性，以减少内存占用。调用该方法后，许多方法无法将无法正常使用，慎用

    • defrost : 解冻一个类，使其可以被修改。如果事先知道一个类会被defrost， 则禁止调用 prune 方法

    • detach : 将该class从ClassPool中删除

    • toClass : 通过类加载器加载该CtClass将修改后的CtClass加载至当前线程的上下文类加载器中，CtClass的toClass方法是通过调用本方法实现。需要注意的是一旦调用该方法，则无法继续修改已经被加载的class

    • writeFile : 根据CtClass生成 .class 文件

  • CtMethod

    • CtMthod代表类中的某个方法，可以通过CtClass提供的API获取或者CtNewMethod新建
    • insertBefore : 在方法的起始位置插入代码；
    • insterAfter : 在方法的所有 return 语句前插入代码以确保语句能够被执行，除非遇到exception
    • insertAt : 在指定的位置插入代码
    • setBody : 将方法的内容设置为要写入的代码，当方法被 abstract修饰时，该修饰符被移除
    • make : 创建一个新的方法

  • CtField

  • CtField代表CtClass的成员变量

  • make: 生成新的成员变量

3注意事项

符号 含义
$0, $1, $2, … $0表示this，$1表示方法第一个参数 …
$args 方法参数数组.它的类型为 Object[]，注意：$arg[0]对应$1
$$ 所有实参。例如, m($$) 等价于 m($1,$2,…)
$cflow(…) cflow 变量
$r 返回结果的类型，用于强制类型转换
$w 包装器类型，用于强制类型转换 ,eg: Integer i = ($w)5;
$_ 返回值
$sig 类型为 java.lang.Class 的参数类型数组
$type 一个 java.lang.Class 对象，表示返回值类型
$class 一个 java.lang.Class 对象，当前正在修改的类

  • 在写入某些对象是要加上包全路径名称

    例如： Date，List，Map 或者自定义的类

    classPool.importPackage(“java.util.List”)

  • 在使用泛型时，泛型定义需要使用/**/将其包括起来

    例如：List 需要写成 List//

  • 某些类型的定义不能使用 引用类型 只能 使用 基本类型

    例如：Float,Long

    不能使用Float.valueOf(),只能使用Float.parseFloat() 转换

    }

public static void addMemberMethod() {
    ClassPool pool = ClassPool.getDefault();
    // 店家目标类搜索路径
    try {
        // 导包
        pool.importPackage("java.util");
        // 导入 android.jar
        pool.appendClassPath("/Users/username/Library/Android/sdk/platforms/android-30/android.jar");
        // 导入目标类所在的路径
      pool.appendClassPath("/Users/username/xxx/IDE/FixBug/lib/eco_pay_classes.jar");
        // 获取目标类
        CtClass ctClass = pool.getCtClass("com.payeco.android.plugin.loading.PayecoPluginLoadingActivity");
        // 解冻
        if (ctClass.isFrozen()){
            ctClass.defrost();
        }
        // 给目标类添加成员方法
        // 注意在使用<>这样的泛型定义 标识时 要使用 /* */ 将其包括起来
        // for-each 不支持，for in 是可以的，暂时不知道什么原因， for(Object o : data){
        //        System.out.println(o);
        //      }
        String methodBody = "public  void sayHello() {\n" +
                "    java.util.List/*<String>*/ data = new java.util.ArrayList();\n" +
                "    data.add(\"张三\");\n" +
                "    data.add(\"李四\");\n" +
                "    data.add(\"王五\");\n" +
                "    for(int i = 0; i< data.size();i++){\n" +
                "       System.out.println(data.get(i)+\" say hello\");\n" +
                "      }\n" +
                "}";
        CtMethod newMethod = CtNewMethod.make(methodBody,ctClass);
        ctClass.addMethod(newMethod);
        ctClass.writeFile("/Users/username/xxx/IDE/FixBug/build");
        ctClass.detach();
    } catch (NotFoundException | CannotCompileException | IOException e) {
        e.printStackTrace();
    }
}

  • 在方法中插入代码

    /**
  * 在方法中插入代码
  */
 public static void insertCodeForMethod() {
     ClassPool pool = ClassPool.getDefault();
     // 店家目标类搜索路径
     try {
         // 导入 android.jar
         pool.appendClassPath("/Users/username/Library/Android/sdk/platforms/android-30/android.jar");
         // 导入目标类所在的路径
       pool.appendClassPath("/Users/username/xxx/IDE/FixBug/lib/eco_pay_classes.jar");
         // 获取目标类
         CtClass ctClass = pool.getCtClass("com.payeco.android.plugin.loading.PayecoPluginLoadingActivity");
         // 解冻
         if (ctClass.isFrozen()){
             ctClass.defrost();
         }
         // 给目标类成员方法 onCreate 添加代码
         CtMethod ctMethod = ctClass.getDeclaredMethod("onCreate");
         ctMethod.insertBefore("System.out.println($0+\"insert  onCreate() before\");");
         ctMethod.insertAfter("System.out.println($0+\"insert  onCreate() after\");");
         // 给c方法指定行插入代码
         CtMethod ctMethod1 = ctClass.getDeclaredMethod("a");
         ControlFlow flow = new ControlFlow(ctMethod1);
         ControlFlow.Block[] blocks = flow.basicBlocks();
         for (ControlFlow.Block block : blocks) {
             String blockIndex = Integer.toString(block.index());
             int p1 = block.position();
             //计算源代码的行号，Source code line position from binary line position
             int pos = ctMethod1.getMethodInfo().getLineNumber(p1);
             System.out.println("blockIndex="+blockIndex+",p1="+p1+",pos="+pos);
             System.out.println("block="+block.clientData);
             if (blockIndex.equals("0")){
                 ctMethod1.insertAt(pos,"System.out.println($0+\"insert  a(\"+$1+\") at line num:"+pos+"\");");
             }
         }
         ctClass.writeFile("/Users/username/xxx/IDE/FixBug/build");
         ctClass.detach();
     } catch (NotFoundException | CannotCompileException | IOException | BadBytecode e) {
         e.printStackTrace();
     }
 }

  • 创建和生成class

    /**
  * 创建一个类
  */
 public static void createPerson() {
     ClassPool pool = ClassPool.getDefault();
     try {
         // 创建一个类
         CtClass ctClass = pool.makeClass("com.payeco.android.plugin.loading.Person");
         // 新增字段 name
         CtField member1 = new CtField(pool.get("java.lang.String"),"name",ctClass);
         member1.setModifiers(Modifier.PRIVATE);
         // 设置初始值
         ctClass.addField(member1,CtField.Initializer.constant("张三"));
         //新增字段 age
         CtField member2 = CtField.make("private int age = 0 ;",ctClass);
         ctClass.addField(member2);
         // 生成getter setter
         ctClass.addMethod(CtNewMethod.setter("setName",member1));
         ctClass.addMethod(CtNewMethod.getter("getName",member1));
         ctClass.addMethod(CtNewMethod.setter("setAge",member2));
         ctClass.addMethod(CtNewMethod.getter("getAge",member2));
         // 添加无参构造函数
         CtConstructor cons = new CtConstructor(new CtClass[]{},ctClass);
         cons.setBody(" { name = \"小明\"; \n" +
                 "          age = 18;  \n" +
                 "        }");
         ctClass.addConstructor(cons);
         // 添加有参构造函数
         cons = new CtConstructor(new CtClass[]{pool.get("java.lang.String"),pool.get("int")},ctClass);
         // $0=this, $1,$2,$3... 代表方法参数
         cons.setBody("{ $0.name=$1;\n" +
                 "        $0.age=$2;}");
         ctClass.addConstructor(cons);
         // 创建一个 sayHello 方法 无参数，返回类型为void
         CtMethod ctMethod = new CtMethod(CtClass.voidType,"sayHello",new CtClass[]{},ctClass);
         ctMethod.setModifiers(Modifier.PUBLIC);
         ctMethod.setBody("{System.out.println($0.getName()+ \" sayHello\");}");
         ctClass.addMethod(ctMethod);
         // 获取接口
         CtClass interfaceI = pool.get("java.io.Serializable");
         // 使代码生成的类，实现 Serializable 接口
         ctClass.setInterfaces(new CtClass[]{interfaceI});
         // 创建实例
         Class clz = ctClass.toClass();
         Object instance = clz.newInstance();
         Method method = clz.getDeclaredMethod("sayHello");
         method.setAccessible(true);
         method.invoke(instance, (Object[]) null);
  
         // 保存class 文件
         ctClass.writeFile("/Users/username/xxx/IDE/FixBug/build");
         ctClass.detach();
     } catch (CannotCompileException | NotFoundException | IOException | NoSuchMethodException | InstantiationException | IllegalAccessException | InvocationTargetException e) {
         e.printStackTrace();
     }
 }

5实战，修复易联支付sdk的crash

情景回放：由于项目的targetSdk升级到29，导致TelephonyManager.getDeviceId() 不可用，抛异常crash，由于沟通之后第三方暂未兼容，尝试自己修复,需要hook相应方法，修改代码逻辑

  1. 使用zip命令解压原始jar或则aar

    # 使用unzip命令，将jar包 解压到指定目录
unzip ./FixBug/lib/eco_pay_classes.jar -d ./FixBug/lib/fix

  2. 使用Javassist 进行字节码修改，修复问题

public static void fixEcoPay() {
    ClassPool pool = ClassPool.getDefault();
    // 店家目标类搜索路径
    try {
        // 导入 android.jar
        pool.appendClassPath("/Users/username/Library/Android/sdk/platforms/android-30/android.jar");
        // 导入目标类所在的路径
        pool.appendClassPath("/Users/username/xxx/IDE/FixBug/lib/eco_pay_classes.jar");
        // 获取目标类
        CtClass ctClass = pool.getCtClass("com.payeco.android.plugin.d.k");
        // 解冻
        if (ctClass.isFrozen()){
            ctClass.defrost();
        }
        // 获取目标method
        CtMethod methodB = ctClass.getDeclaredMethod("b",new CtClass[]{pool.get("android.content.Context")});
        String fixCode = "if(true){\n" +
                "\treturn \"\";\n" +
                "}";
        methodB.insertBefore(fixCode);
        ctClass.addField(CtField.make("private static String aaa = \"12345\";",ctClass));

        CtMethod methodC = ctClass.getDeclaredMethod("c",new CtClass[]{pool.get("android.content.Context")});
        methodC.insertBefore(fixCode);
        ctClass.writeFile("/Users/username/xxx/IDE/FixBug/build");
        ctClass.detach();
    } catch (NotFoundException | CannotCompileException | IOException e) {
        e.printStackTrace();
    }
}

  1. 将修改之后的class对第一步解压之后的class文件进行替换，并使用zip命令重新生成jar包

    # 使用zip -r 将解压缩之后的所有class文件，压缩到eco_pay_classes_fix.jar
zip -r ../eco_pay_classes_fix.jar ./*

  2. 将修复之后的jar在工程中进行替换即可
